## Design Principles

### Core Principles
1. **Simplicity**: Clear, uncluttered interface focused on the task
2. **Feedback**: Immediate visual feedback for all user actions
3. **Progressive Disclosure**: Show information when needed, not all at once
4. **Consistency**: Uniform design patterns throughout the app
5. **Accessibility**: WCAG 2.1 AA compliance

### Visual Hierarchy
1. Primary action: Upload resume (most prominent)
2. Secondary: View results
3. Tertiary: Export/share options

### Interaction States

```
1. Default State (Landing)
   - Upload area: Visible, prominent
   - Call-to-action: "Upload Resume"
   - Info: Feature highlights

2. Hover State (Upload)
   - Border: Highlight color
   - Cursor: Pointer
   - Background: Subtle color change

3. Drag Over State
   - Border: Solid, bright color
   - Background: Highlighted
   - Text: "Drop file here"

4. Uploading State
   - Progress bar: Animated
   - Upload area: Disabled
   - Text: "Uploading... X%"

5. Processing State
   - Step indicator: Active
   - Progress bar: Animated
   - Status messages: Updating
   - Cancel button: Visible

6. Success State (Results)
   - Results: Displayed
   - Animations: Subtle entrance
   - Actions: Export, New Analysis

7. Error State
   - Error message: Clear, actionable
   - Retry button: Prominent
   - Previous data: Cleared
```

- 2-column grid for results

- Collapsible sections
- Larger touch targets

- Side-by-side comparisons
- Hover states
- Detailed visualizations

## Animation & Transitions

### Principles
1. **Purposeful**: Animations should guide user attention
2. **Subtle**: Not distracting or overwhelming
3. **Fast**: Duration < 300ms for most transitions
4. **Responsive**: Respect prefers-reduced-motion

### Key Animations

**Page Transitions**
```
Duration: 200ms
Easing: ease-in-out
Effect: Fade in + slight slide up (10px)
```

**Score Reveal**
```
Duration: 1000ms
Easing: ease-out
Effect: Number counting from 0 to final score
       + Circular progress drawing
```

**Chart Animations**
```
Duration: 800ms
Easing: ease-out
Effect: Bar charts grow from 0 to final value
        with 50ms stagger between bars
```

**Card Entrance**
```
Duration: 300ms
Easing: ease-out
Effect: Fade in + scale from 95% to 100%
Stagger: 50ms between cards
```

**Hover Effects**
```
Duration: 150ms
Easing: ease-in-out
Effect: Smooth color transitions
        + Subtle elevation change (shadow)
```

## Design System Summary

This design system provides:
- **Consistency**: Unified visual language
- **Scalability**: Reusable components
- **Accessibility**: WCAG 2.1 AA compliant
- **Responsiveness**: Mobile-first approach
- **Maintainability**: Well-documented patterns

All components follow Material Design principles with custom branding and optimizations for the resume analysis use case.
